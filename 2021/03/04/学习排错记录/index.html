一方面记录一下常见的错误以备不时之需，另一方面也要提升自己的检索排错能力

这两天实习，发现项目框架都没学，疯狂看视频写代码，然后遇到超级多问题。。。有些还重复遇到，并且解决方法还不一样

所以在此记录下我写代码中遇到的Bug和找到的解决方法

【问题】：运行项目出现java: 程序包org.springframework.web.bind.annotation不存在 的错误。

【解决方法】：

  • 网上方案都是说在Setting中的maven下勾选上Runner中的第一个选项，但是我试了没用。。。
  • 也有说maven刷新问题，在右侧maven中clean再install。试了还是没用。。。
  • 最后我换了低版本的maven依赖，然后再试发现可以了

【问题】：Navicat查看数据库表的时候出现Error too many connections

【解决方法】：

  • 想命令行登录mysql看一下啥情况，结果也说too many connections，然后查了一下，navicat还连接着，就可以直接用navicat查询，网上找到的命令行查看最大连接数：show VARIABLES like "%max_connections%"。连接数只有20，所以超过连接了。。

【问题】：运行npm run serve命令时报错npm ERR! code ELIFECYCLE npm ERR! errno 1

【解决方法】：

  • 我另一个VSCode也在跑一个前端，所以我猜可能是端口占用了，把那个关闭，没有解决，检查了一下发现两个项目没用同一个端口，那就是别的问题了
  • 检查看提示，node_Modules缺东西，我直接全删了，然后npm install就好了。

【问题】：虚拟机选桥接模式不能连接外网

这个真的奇怪，我想着搭个虚拟机本地部署一个fastdfs，然后按照教程步骤来，网络选桥接模式时不能ping到外网，ifconfig也看不到ip地址

【解决方法】：

  • 第一个：在网络编辑器中，设置桥接模式，选择与宿主机相同的网卡；使用dhclient分配IP地址（我用了这个指令还是没IP地址）；按照他给的文件去编辑ifcfg-ens33；然后重启网络。【没用】
