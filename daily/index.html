2020-10-09 环形链表

思路：双指针的典型应用，在labuladong的pdf上看到过，设置慢指针slow走一步快指针fast走两步，每一轮快慢指针间隔+1，这样如果有环，最后fast一定会追上slow。

  • 因为先进行while循环的判断，所以如果slow和fast都初始化为head就不会进入循环体了
  • 可以改为do-while方法，这样都设置为head就没问题

2020-10-10 环形链表 II

思路：算是昨天题目的延续，不过今天要返回的是环的入口。即先判断环；再找到环入口。同样考虑一下快慢指针

  • 无环情况：fast到终点结束

  • 有环情况：fast会追上slow。假设链表有a+b个节点【表头到入口a个，环b个】

    • 第一次相遇时，slow若走了n步，fast一定走了2n步；多走的为一个环的长度b即b=n。设相遇点到入口为x，则入口到表头为a = n-x；而fast此时到入口的距离b-x == n-x，所以只需要将slow再次置为head，然后二者每次一步就能碰面

      【当然第一次相遇fast也可能走了n圈了】


2020-10-11 分割等和子集

思路：第一反应是先排序，然后从后往前添加到子集中来判断，如果比剩余之和大就false，否则就继续从前往后加，这样做测试数据还能过，但是很容易举反例3,3,3,4,5应该是取9，按照上面的思路会是5+3+3….即使换成每个数都完整判断也不行。。。

看了题解用【01背包】求解，题目要求从数组中挑选元素确实就跟01背包一样！利用动态规划将问题求解转化为对其子问题的求阶。思路如下：

  • 每个数有选和不选两种方案，就跟01背包一样。用一个dp[i][j]数组表示状态——表示从数组[0...i]中挑选正整数使得和恰好为j。
    • 选nums[i]。表示[0...i-1]子区间中找到的元素和为j-nums[i]
    • 不选nums[i]。表示[0...i-1]子区间中已经找到元素和为j
  • 状态转移方程：dp[i][j] = dp[i - 1][j] or dp[i - 1][j - nums[i]]
    • 由状态转移方程–>nums[i]<=j
    • 前i-1个元素已经能找出和等于j的解时，dp[i][j] = dp[i - 1][j]
    • 第i个元素恰好等于j时，dp[i][j]=true
    • 第i个元素小于j时，转化为对子问题从[0...i-1]元素中找和等于j-nums[i]的解
    • 所以题目要找的就是dp[数组元素个数][数组元素总和的一半]的值

【注】1、因为要将数组分成两个和相同的部分，所以总和必定是偶数

【优化1】设置剪枝

  • 对于dp[i][0]本来应该设为false；但是状态转移中有dp[i-1][j-nums[i]]所以当nums[i]==j时它的子问题就成了dp[i-1][0]，即数组中恰好有一个数nums[i]==target，即剩下的数和等于它，所以设置dp[i][0]=0操作上可行的
  • 状态转移方程是or，即只要其为真，这一列的值都为真

【优化2】空间优化

  • 对于状态空间，当前状态dp[i][j]只参考了dp[i-1][j]和dp[i-1][j-1]两个，相当于自底向上的递推，所以这完全可以用一维空间来代替（类似北大的慕课程序设计与算法（二）算法基础 动态规划讲的）
  • 「从后向前」 写的过程中，一旦 nums[i] <= j 不满足，可以马上退出当前循环，因为后面的 j 的值肯定越来越小，没有必要继续做判断，直接进入外层循环的下一层。相当于也是一个剪枝，这一点是「从前向后」填表所不具备的。

动态规划真的太难了。。。即使看题解也整了半天才明白。题刷太少了

2020-10-13 两两交换链表中的节点

这是之前做过的一道题。。并不是[1,2,3,4]输出[4,3,2,1]而是两两交换，所以输出应该是[2,1,4,3]，题目要求是实际交换节点

思路：常规就是递归，比较好想，要注意三点：递归结束条件、返回值、调用函数执行过程；

  • 递归解决：两两交换，那么交换后的1指向的就是交换后的4，如果是[1,2,3,4,5,6]那就是1指向4，3指向6这样，即前一个的next指向的是后一个的后面交换的结果，这就可以用递归。这种思路比较好理解；
  • 可以递归就可以迭代：用一个dummy节点存储，per表示当前指向的节点，每次交换的是per后面的两个节点，例如per->node1->node2交换完后，变成per->node2->node1然后再per=node1，进行下一轮

2020-10-15 填充每个节点的下一个右侧节点指针

思路：题目要求每一层都是指向右边，最右边的指向null，对于root节点1没有兄弟节点，所以看它的子节点，root.left.next = root.right，而到了第二层，左子树2有兄弟右子树3，要完成将两棵树连在一起，就需要将左子树的右子节点和右子树的左子节点相连，当前左子树的root是2，那么对应的即root.right.next=root.next.left。然后递归遍历左右子树。

【注】对应的也可以用迭代实现

2020-10-22 划分字母区间

其实这题一开始没看懂。。。看了别人的题解才明白他想求啥。。。

思路：遍历字符串，统计每一个字符最后出现的位置，然后看这个区间里的其他字符是否都在区间内，如果区间内的字符在外面还有那就要更新区间，如果没有就不用，然后遍历到右边下标等于位置值处，就是切割点。其实在找到切割点后，问题就转变为了不相交区间问题，所以是贪心思想解决

2020-10-24 视频拼接

思路：找到最小的区间个数能覆盖要求的长度，可以用区间贪心的方式，也可以动规解决

  • 【贪心思路】：贪心的思想就是每个子问题都取最优情况，对应本题就是子区间尽量取更长的。那么对于T=10即长度为10的目标，我们可以记录0-10每个位置能到的最远的右端点记为maxEnd[10]，然后遍历clips，完成maxEnd的初始化。
    • 定义pre记录上次最大结束位置；定义last记录当前遍历到的最大结束位置；定义ans记录结果
    • 当前元素==本区间最大元素：表示区间断开，无法连续到后面的位置，return -1
    • 当前区间==上一个区间最大结束位置：到达了上一个满足条件区间的结束位置，此时last为当前的最大结束位置，更新满足条件的区间集合

2020-10-25 数组中的最长山脉

思路：首先要理解题目所说的”山脉“的概念，就是一个序列，左侧到山顶严格递增，山顶到右侧严格递减，例如[1,4,7,3,2]。

  • 利用山脉的特点，依次判断每个节点能作为山顶拓展多远，并更新结果。
    • 用left[i]表示A[i]能向左拓展的元素数，由递推关系可以得到：left[i]=left[i-1]+1；如果左侧元素比当前大，则对应的left[i]=0
    • 同理right[i]表示右侧。也是right[i]=right[i-1]+1 OR right[i]=0
    • 满足left和right都大于0才可看作山脉，对应的长度即为left[i]+right[i]+1
  • 还可以用双指针！官方题解判断过多，另外的一篇题解很简洁！
    • increase表示左侧，decrease表示右侧，通过i++来同步；

2020-10-31 O(1) 时间插入、删除和获取随机元素 - 允许重复

思路：一看是困难就慌了。。。只想到是不是hash之类的。。。

  • 官方题解有点难懂，另一篇采用list+hashmap。题目要求O(1)时间前提下：插入时要判断是否已存在；获取的概率要与它在集合中的数量线性相关
  • 例如初始[]插入1得到[1]返回true；再插入1得到[1,1]返回false；插入2得到[1,1,2]返回true；获取1的概率为2/3，获取2的概率为1/3；
    • list删除尾元素才是O(1)，所以可以将要删除的元素和队尾元素交换
    • Map存的key-value中可能出现多对一，所以key也要用集合封装，由于同一个值虽然有可能有多个key但是肯定不会有相同的key，所以可以用Set存储
